Data Gathering

Data source

  1. The Twitter Archive Enhanced file was downloaded from Udacity directly, then read into a dataframe archive_df
  1. Image Predictions File was read into img_df from the URL provided.
  1. Data on the json objects of the tweets appeared in archive_df was extracted from the Twitter API, and read into json_df.

Data Assessing

archive_df was first assessed, resulting in some data issues:

  1. Dog stages can be represented by 1 and 0 instead of the name of the dog stage.
  1. Some dogs are in more than one stage
  1. Some dogs have weird or no names e.g. old, officially, mad, such, a
  1. The maximum numerators and denominator are way too large. But the rating system seems to be arbitrary anyways.
  1. The denominator should be 10, but there are 16 with bizzare denominators.

The above issues were translated into next steps for data cleaning:

  1. change all ids that are in float to int
  1. change timestamp from string to datetime
  1. change retweeted_status_timestamp from string to datetime
  1. change all dog types from string to booleans (doggo floofer pupper puppo)
  1. I'll leave dog names as they are for now.
  1. Change denominator to be 10

Data Cleaning

  1. Whether or not it is float or int does not detriment the data analysis process, therefore IDs are kept as float instead of changing to integer: 'in_reply_to_status_id', 'in_reply_to_user_id', 'retweeted_status_id', 'retweeted_status_user_id'
  1. Dates are changed from string to timestamps: retweeted_status_timestamp and timestamp
  1. Dog stages are changed from string to 0s and 1s: 'doggo', 'floofer', 'pupper', 'puppo'
  1. Tweets with no image are removed by inner merging archive_df with img_df.
  1. Retweets are removed by dropping rows with a retweeted_status object
  1. Tweets beyond August 1st, 2017 does not exist
  1. Extraneous columns are removed: 'in_reply_to_status_id', 'in_reply_to_user_id','retweeted_status','retweeted_status_id','retweeted_status_user_id','retweeted_status_timestamp','id'
